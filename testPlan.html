Homeaglow Purchase Flow Test Plan

1. Introduction

Objective: Ensure a seamless customer journey from clicking the ad to scheduling an appointment on Homeaglow.

Scope: Includes ad click redirection, website interactions, and purchase completion.

2. Test Items

  • Ad Redirection Functionality
  • Eligibility Check based on Zip Code
  • Voucher Selection and Purchase Process

3. Features to be Tested

  • Redirection from various advertising platforms to the Homeaglow website
  • Interactive elements and information accuracy on the deals page
  • User information input validation and functionality
  • Payment processing with multiple methods

4. Features Not to be Tested

  • Areas outside the purchase flow, such as user profile management or scheduling.
  • Emails/texts - these are blocked on staging and don't need to be included.
  • Apple+Google Pay

5. Testing Approach

Functional Testing

This involves verifying the functionalities of the website according to the requirements. It includes testing all user interactions, data processing, and validations within the purchase flow.

Test cases:

Usability Testing

Focuses on the user experience to ensure the website is intuitive and easy to navigate. This includes assessing the design, layout, and overall user journey through the purchase process.

Performance Testing

Ensures that the website performs well under various conditions. This includes testing load times, responsiveness, and the website's ability to handle a high volume of users simultaneously.

Accessibility Testing

Checks the website's compliance with accessibility standards to ensure it is usable by people with disabilities. This includes testing for screen reader compatibility, color contrast, and keyboard navigation.

Cross-Browser and Device Compatibility

The selection of browsers and devices is based on the target user demographic and popular usage statistics. We aim to cover a diverse range of platforms to ensure the majority of our users have a seamless experience.

Desktop Browsers: Latest version of Chrome, Safari, Firefox, Edge.

Mobile Browsers: Safari on iOS and Google Chrome on Android. We will test on the latest and previous versions of iOS and Android.

Devices: Desktops, laptops, tablets, and mobile devices.

Browser Settings: enabling/disabling JavaScript

Test Automation

We will utilize the Playwright framework with TypeScript for automated testing to cover the most common and high-impact user journeys, repetitive and regression testing, data validation, and high-level performance testing.

6. Entry/Exit Criteria

Entry Criteria

  • Completion of development for the purchase flow features.
  • Availability of the test environment, including setup for various browsers and devices.
  • Initial test cases and test scripts are prepared and reviewed.

Exit Criteria

  • All critical and major defects fixed and retested successfully.
  • Achievement of predefined test coverage thresholds.
  • Completion of regression testing without any major issues.
  • Approval from the QA lead or project manager confirming that testing objectives are met.
